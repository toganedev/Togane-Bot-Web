📘 コマンド一覧

/ping

/ping

BotとDiscord APIの応答速度（レイテンシ）を確認します。Bot Latency / API Latency / WebSocket Ping を embed形式で表示。

/verify

/verify

指定したロールをユーザーに付与する認証機能です。2つの認証方法から選択可能です：

  • ✅ ワンタッチ認証：ボタンを押すだけでロールを付与
  • 🧠 計算認証：簡単な計算問題に正解するとロール付与

埋め込みタイトル、説明文、画像、ボタンスタイルをカスタマイズ可能。デバイスからアップロードした画像を使えます。

/welcome-setting

/welcome-setting

サーバーへの新規参加者の入室ログを設定・削除するためのコマンドです。

  • /welcome-setting set：入室ログの送信チャンネルを指定
  • /welcome-setting remove：設定済みの入室ログを削除（削除確認あり）

設定後、管理者のDMにJSON形式で送信され、手動で welcomeChannels.json に反映されます。

/ticket

/ticket

サポートが必要なユーザーのために、個別の「チケット」チャンネルを作成します。

  • 🎫 チケット作成ボタン：指定カテゴリ内にチャンネルを自動生成し、対応ロールに通知
  • 🗑️ 削除ボタン：管理者のみが使用可能。チャンネルを削除し、ログが欲しい場合はtogane_devまで連絡を

カスタムIDに role や cat を指定することで、呼び出し対象ロールやカテゴリ設定が可能です。

/togane-server-template

/togane-server-template

固定のToganeサーバーテンプレートリンクを、コマンド実行者のDMに送信します。

  • DMにはEmbed形式でリンクを送信
  • 実行者のDMが閉じている場合はエラー通知
  • テンプレートリンクは管理者が設定

例: 「📄 Togane サーバーテンプレート」というタイトルのEmbedに、
[テンプレートを開く](https://discord.com/template/xxxxx) のリンクが表示されます。

/role-info

/role-info

サーバー内の全ロールを、権限順（上位ロールから順）に一覧表示します。

  • ロール名の横に、以下の主要権限の有無を表示：
    • 管理者権限
    • BAN権限
    • タイムアウト権限
    • KICK権限
  • 権限の有無は ✅（あり）/ ❌（なし）で表示
  • 結果はコードブロック形式でEmbedに出力
  • 管理者が実行すると、サーバー管理に役立つロール情報をすぐ確認可能
/clear

/clear

指定条件に基づいてメッセージを一括削除します。ユーザー指定・日数指定・キーワード指定が可能で、実行結果はEmbedでサーバー作成者に通知されます。

  • 指定可能な条件：
    • ユーザー指定（そのユーザーのメッセージのみ削除）
    • 日数指定（指定日数分だけ遡って削除）
    • キーワード指定（そのキーワードを含むメッセージのみ削除）
  • 削除数や条件をEmbedでサーバー作成者にDM送信
  • メッセージ管理権限（ManageMessages）が必要
  • 古いメッセージ（14日以上前）はDiscordの制限により削除不可
  • 条件未指定の場合は直近のメッセージから削除
/ban

/ban

指定したユーザーをサーバーからBANし、BAN通知をサーバー管理者へDMで送信します。

  • ユーザー指定方法：
    • サーバー内のユーザーを直接選択
    • ユーザーIDを指定（サーバー外のユーザーも可）
  • BAN理由を必須で指定
  • 処理完了後、以下の情報をEmbedでサーバー管理者へDM送信：
    • BAN実行者
    • BAN対象ユーザー（タグとID）
    • BAN理由
    • BAN日時（日本時間・UTC+9）
  • 管理者は通知からBAN履歴を即座に確認可能
/unban

/unban

指定したユーザーのBANを解除し、解除通知をサーバー管理者へDMで送信します。

  • ユーザー指定方法：
    • ユーザーIDを指定（サーバー外のユーザーも可）
  • 解除理由を必須で指定
  • 処理完了後、以下の情報をEmbedでサーバー管理者へDM送信：
    • 解除実行者
    • 解除対象ユーザー（タグとID）
    • 解除理由
    • 解除日時（日本時間・UTC+9）
  • 解除前にBANリストを確認するため、既に解除済みのユーザーには「現在BANされていません」と通知
  • 管理者は通知から解除履歴を即座に確認可能
/kick

/kick

指定したユーザーをサーバーからキックします。キック通知は管理者DMと実行チャンネル両方に送信されます。

  • 対象者・実行者・理由・日時（日本時間・UTC+9）をEmbedで表示
  • BOTの権限不足や対象者がサーバーにいない場合はエラー通知
  • 管理者のDMが閉じている場合は実行チャンネルのみ通知
/timeout

/timeout

指定したユーザーを一定時間ミュート（タイムアウト）します。タイムアウト中はメッセージ送信やボイス参加ができません。

  • 時間指定は以下から選択：
    • 1分 / 5分 / 10分 / 30分 / 1時間 / 5時間 / 10時間 / 1日 / 7日
  • 管理者DMと実行チャンネルに、設定時間・開始時間・解除予定時間（日本時間・UTC+9）をEmbed送信
  • 既にタイムアウト中の場合は、残り時間を秒単位で表示
  • 対象者・実行者・理由を記録
/untimeout

/untimeout

指定したユーザーのタイムアウトを解除します。

  • 対象がタイムアウト中でない場合は「現在タイムアウトされていません」と通知
  • タイムアウト中の場合は残り時間を「日・時間・分・秒」に変換して表示し、確認後に解除
  • 解除後、管理者DMと実行チャンネルに解除通知（対象者・実行者・解除日時）をEmbed送信
/font

/font

入力されたテキストを指定のフォントスタイルに変換してコードブロックで送信します。

  • 対応フォント：
    • bold-italic：太字イタリック
    • cursive：筆記体
    • old-english：ブラックレター（古風な書体）
    • cool-text：丸文字
  • 英字はUnicode装飾文字、日本語は全角変換で装飾
  • 例：/font style:cool-text text:Hello 世界
/neko

/neko

画像を送信します。画像は外部APIから取得。

  • 1回実行するごとに違う画像が表示
  • 画像はEmbed形式で送信
  • 例：Hな画像など様々な写真
/play

/play

指定した音楽をボイスチャンネルで再生します。URLまたはキーワード検索が可能。

  • 対応形式：BOT製作者がダウンロードした音源
  • 再生中に同じコマンドを使うとキューに追加
  • ボイスチャンネルに参加している必要あり
  • 例：/play 曲名
/stop

/stop

再生中の音楽を停止し、キューをすべてクリアします。

  • 管理者または音楽再生権限を持つユーザーのみ使用可能
  • 停止後は/playで再度再生可能
/skip

/skip

再生中の曲をスキップして、次の曲へ移ります。

  • キューが空の場合は停止します
  • ボイスチャンネルに参加している必要あり
  • 例：次の曲をすぐ聴きたいときに便利
/banner

/banner

指定したユーザーのバナー画像を表示します（未設定の場合はお知らせ）。

  • 対象を指定しない場合は自分のバナーを取得
  • サイズは最大2048pxで表示
  • ユーザーIDと共にEmbedで送信
/avatar

/avatar

指定したユーザーのアバター画像を表示します。

  • 対象を指定しない場合は自分のアバターを取得
  • 動くアバターにも対応（GIF表示）
  • 最大1024pxで高画質表示
